### Why Another Vector Database?

The vector database with the best DX is undoubtedly ChromaDBâ€”you should probably just use ChromaDB for most usecases.

But here's the thing: ChromaDB's TypeScript SDK doesn't support arrays as metadata values. This limitation makes a whole class of features (like ACL-based filtering) harder than it needs to be.

The other reason? **WiredTiger.** A powerful NoSQL engine I discovered during my weekly tinkering sessions on GitHub. Reading its source code was my gateway drug into databases and data systems. So I thoughtâ€”why not combine all of these cool primitives into one product?

---
